# Reference Analysis

## 1. 参考资产

当前 `ref/` 下有两个参考仓库：

- `ref/llcom`
- `ref/Wu.CommTool`

## 2. llcom 的可复用价值

### 核心价值

- 串口为核心的高自由度调试工作台
- Lua 脚本深入收发前后处理
- 快捷发送和多页模板管理
- TCP/UDP/TCP SSL/MQTT/WinUSB/串口监听/曲线等工具型能力齐全
- 通道抽象已经出现雏形：`uart`、`socket-client`、`mqtt`、`winusb`

### 适合继承的方向

- “脚本和规则是一等公民”的思路
- 原始数据与展示分离
- 快速调试体验
- 多工具同平台共存

### 不建议直接继承的部分

- 大量全局状态
- 入口窗口承担过多职责
- 页面与逻辑耦合较深

## 3. Wu.CommTool 的可复用价值

### 核心价值

- 模块化组织更清晰
- 工业协议工作流更完整
- Modbus RTU 已经接近“可交付流程”
- 配置导入导出、样例配置、设备监控这些产品化意识更强

### 适合继承的方向

- 模块化导航结构
- 功能按协议域拆分
- 配置文件驱动
- 设备监控和自动应答模型

### 不建议直接继承的部分

- 直接照搬 WPF/Prism 结构到新项目没有意义
- 部分功能还处于半完成或不一致状态
- 传输抽象仍然不够统一

## 4. ProtoLink 的吸收策略

ProtoLink 采用以下合并原则：

- 借 `llcom` 的自动化与通道思维
- 借 `Wu.CommTool` 的模块化与工业协议流程
- 不直接 fork 任一参考仓库作为主线
- 所有核心概念重新建模

## 5. 功能吸收矩阵

| 能力域 | llcom | Wu.CommTool | ProtoLink 策略 |
| --- | --- | --- | --- |

| Modbus RTU | 中 | 强 | 首版主攻工作流 |
| Modbus TCP | 弱到中 | 中 | 第二阶段 |
| MQTT | 中 | 中 | 传输核心能力 |
| TCP/UDP | 中 | 中 | 传输核心能力 |

| 配置导入导出 | 中 | 强 | 统一规范 |
